#### 4. ExecWindows — RC2104 DS_SETFONT 오류
- **행위**: `Windows.vcxproj`의 RC 컴파일러에 `$(WindowsSDK_IncludePath)` 추가 시도
- **원인**: 프로젝트명 충돌(`Windows.h`와 시스템 헤더 충돌)로 인한 오류
- **결과**: 프로젝트명을 `ExecWindows`로 변경 후 재생성하여 해결

#### 5. ExecAndroid — 빌드 오류 5단계 수정
**① JAVA_HOME 미설정**
- 원인: JDK가 설치되지 않았거나 환경변수 미설정
- 결과: JDK 17 설치 및 `JAVA_HOME`, `PATH` 환경변수 설정으로 해결

**② Android SDK 라이선스 미동의 (권한 문제)**
- 원인: SDK가 `C:\Program Files (x86)\`에 설치되어 일반 권한으로 라이선스 파일 저장 불가
- 결과: **관리자 권한** CMD에서 `sdkmanager --licenses` 실행으로 해결

**③ android-33, build-tools;33.0.3 미설치**
- 원인: VS 설치 시 android-35/36만 설치되고 33은 없었음
- 결과: 관리자 권한 CMD에서 `sdkmanager "platforms;android-33" "build-tools;33.0.3"` 설치로 해결

**④ Gradle 설정 불일치**
- 원인: `build.gradle.template`의 `$(GradlePlugin)` 변수가 VS 내부에서 `7.2.0`으로 주입되나 compileSdk 33 미지원. `$(AndroidBuildToolsVersion)`은 `36.0.0`으로 주입되어 불일치. `local.properties` 없어 Gradle이 SDK 경로를 모름. Java 호환성 버전이 `VERSION_1_7`로 현재 JDK에서 deprecated
- 결과: 아래 파일들 수정
    - `build.gradle.template`: `$(GradlePlugin)` → `7.4.2` 하드코딩
    - `app/build.gradle.template`: `$(AndroidBuildToolsVersion)` → `33.0.3`, `VERSION_1_7` → `VERSION_11`
    - `local.properties` 신규 생성: `sdk.dir=C\:\\Program Files (x86)\\Android\\android-sdk`

**⑤ ExecAndroid.java UTF-8 BOM 오류**
- 원인: VS가 Java 파일을 UTF-8 BOM 형식으로 생성. Java 컴파일러는 BOM을 허용하지 않음
- 결과: BOM 제거 후 재저장으로 해결
