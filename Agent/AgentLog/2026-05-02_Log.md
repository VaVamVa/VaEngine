| `LPCSTR` ↔ `LPCWSTR` 타입 오류 | UNICODE 미정의로 `WNDCLASSEX`가 ANSI 버전으로 해석 | 동일 |
| `reinterpret_cast` 경고 | `int` → 포인터 직접 캐스팅 | `static_cast<INT_PTR>` 중간 단계 추가 |

### 요약 내용

#### 1. RHI 인터페이스 설계 확정

| 인터페이스 | 역할 |
|---|---|
| `IRenderDevice` | GPU 초기화 진입점 + RHI 객체 Factory |
| `ISwapChain` | Present / Resize / BackBufferIndex |
| `ICommandQueue` | ExecuteCommandLists / Signal / Wait |
| `IFence` | Signal / Wait / GetCompletedValue |
| `ICommandList` | 렌더링 명령 기록 (추후 구체화) |

**핵심 설계 결정:**
- `IRenderDevice::Initialize()` — GPU만 초기화 (윈도우 정보 불필요)
- `FNativeDisplayInfo` — `CreateSwapChain(desc)` 시점에 전달 (멀티 윈도우 대응)
- Create 함수 인자 — Descriptor 구조체로 전달 (시그니처 안정성)
- `IRenderDevice` 자체 생성 — `namespace RHI::CreateRenderDevice()` 팩토리

#### 2. 공통 구조체 작성

| 파일 | 내용 |
|---|---|
| `Engine/Public/Common_Display.h` | `NativeDisplayInfo` (EPlatform, Handle, Display) |
| `Engine/Public/Common_RHI.h` | `SwapChainDesc`, `CommandQueueDesc`, `CommandListDesc` |

- `IExecute.h`의 `FNativeDisplayInfo` 제거 → `Common_Display.h` include로 교체
- `uintptr_t RawData` (기존 `uint32_t` → 64bit 포인터 대응)

#### 3. RHILoader — 팩토리 진입점 작성

- `Engine/Public/RHI/RHILoader.h` — `namespace RHI { CreateRenderDevice(); }` 선언
- `Engine/Private/RHI/RHILoader.cpp` — `#if USE_DX12 / USE_VULKAN` 분기 구현

#### 4. DX12 공통 헤더 작성

- `Engine/Private/RHI/DirectX/Common_DirectX.h`
- `<wrl/client.h>` (ComPtr), `<dxgi1_6.h>`, `<d3d12.h>`, `<d3dx12.h>` 포함
- DX12 전용 헤더를 Private 내부에 격리 → Public 인터페이스 오염 방지

#### 5. DirectX-Headers ThirdParty 통합

- `VaEngine/ThirdParty/DirectX-Headers/` 에 배치
- `Engine/CMakeLists.txt` — `USE_DX12` 블록에 include 경로 추가:
  ```cmake
  target_include_directories(Engine PRIVATE
      "${CMAKE_SOURCE_DIR}/ThirdParty/DirectX-Headers/include/directx"
  )
  ```

#### 6. CMakeLists.txt 주요 변경

| 항목 | 내용 |
|---|---|
| Public 헤더 등록 | `FILE_SET HEADERS` 방식으로 전환 (IDE 가시성 + IntelliSense) |
| `USE_DX12` 컴파일 정의 | `target_compile_definitions(Engine PRIVATE USE_DX12)` 추가 |
| DX12 소스 경로 | `Private/RHI/DirectX/*.cpp` |

**발견된 문제:** CMake 변수 `USE_DX12`와 C++ 전처리기 `#if USE_DX12`는 별개. `target_compile_definitions` 없이는 Rider IntelliSense 및 `#if` 블록이 비활성으로 표시됨.
