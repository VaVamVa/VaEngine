**BakeTransformsMap 알고리즘**:
1. 모든 클립의 maxFrameCount 계산 (짧은 클립은 마지막 프레임으로 패딩)
2. 각 clip × frame × bone에 대해:
   - SRT → local Matrix4x4 (`Scale * RotationQuat * Translation`)
   - 부모 체인으로 globalTransform 계산 (topological order 가정)
   - `finalSkinningMatrix = offsetMatrix * globalTransform`
3. Texture2DArray 업로드: Width=boneCount×4, Height=maxFrameCount, ArraySize=clipCount

**Update 흐름**:
```
AnimController::Update(dt, clips)
  → TweenFrameDesc[] 갱신
  → tweenBuffer->Upload(frames)
```

### 2. ImportTool — clip 1개만 생성되는 문제 수정 (3개 버그)

**버그 1: Inspect() bind pose 오분류**

- 원인: `scene->mNumAnimations > 0` 조건이 bind pose 트랙(Mesh FBX 내 duration ~0초)도 true로 분류
- 수정: duration threshold 적용 → `anim->mDuration / tps > 0.5` 초인 경우만 hasAnimations = true

**버그 2: ConvertSkinned() animation-only FBX 지원 안 됨**

- 원인: `boneNameSet`을 mesh bones에서만 수집 → animation-only FBX는 mesh 없음 → 빈 set → 함수 조기 return
- 수정: `boneNameSet.empty()` 인 경우에만 animation channels에서 보강 (조건부, mesh FBX의 본 수 오염 방지)

```cpp
if (boneNameSet.empty())
{
    for (uint32_t a = 0; a < scene->mNumAnimations; ++a)
        for (uint32_t c = 0; c < scene->mAnimations[a]->mNumChannels; ++c)
            boneNameSet.insert(scene->mAnimations[a]->mChannels[c]->mNodeName.C_Str());
}
```

**버그 3: Main.cpp 분기 순서 오류**

- 원인: `!hasBones` 를 먼저 체크 → animation-only FBX(bone 없음)가 정적 메시 경로로 라우팅
- 수정: `hasAnimations` 를 먼저 체크 (hasBones 여부 무관)

```cpp
if (traits.hasAnimations)      → clip 추출 (.clip)
else if (traits.hasBones)      → skinned mesh 추출 (.smesh)
else                           → 정적 메시 추출 (.mesh)
```
