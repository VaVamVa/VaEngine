- 위치: `Platforms/ExecWindows/Private/Execute.h/.cpp`
- **소유권 확립**: `IRenderDevice`, `ICommandQueue`, `ISwapChain`, `IFence`를 `unique_ptr`로 소유.
- **초기화 흐름**: `OnInitialize`에서 Factory를 통해 디바이스 생성 → 초기화 → 큐/펜스/스왑체인 순차 생성 완료.
- **검증**: `OnInitialize` 성공 후 검은 화면 출력을 통해 초기화 성공 확인 (디버그 레이어 에러 없음).

#### 3. 주요 기술 결정 및 해결

| 항목 | 내용 |
|---|---|
| 리소스 소유권 | `IRenderDevice`(공장)가 아닌 `Execute`(사용자) 레이어에서 관리 (확장성 및 1:N 대응) |
| 스마트 포인터 운용 | `unique_ptr::get()`을 통해 소유권 이전 없이 하위 객체(SwapChain 등) 생성 시 포인터 공유 |
| 인코딩 경고 해결 | CMake `add_compile_options(/utf-8)` 추가 → C4819 경고 전역 제거 및 한글 보존 |
| 교차 API 호환성 | `ICommandQueue`와 `IFence`(Timeline Semaphore 대응) 설계가 Vulkan에서도 유효함을 확인 |
