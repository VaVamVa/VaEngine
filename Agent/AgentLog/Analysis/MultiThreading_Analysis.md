# Multi-threading Technique Analysis (멀티스레딩 기법 분석)

프로젝트 내의 멀티스레딩은 주로 비동기 작업 처리와 성능 최적화 실험을 위해 사용됩니다.

## 1. 표준 C++ 스레드 활용 (`01_ThreadDemo.cpp`)
- **Thread 관리**: `std::thread`를 사용하여 병렬 작업을 생성합니다. `bind`를 통해 클래스 멤버 함수를 스레드 엔트리 포인트로 사용합니다.
- **동기화**: `std::mutex`를 사용하여 공유 자원(예: `counter`)에 대한 경쟁 상태(Race Condition)를 방지합니다.
- **제어**: `join()`을 통한 스레드 종료 대기와 `detach()`를 이용한 백그라운드 실행 기법이 포함되어 있습니다.

## 2. 성능 측정 및 타이머
- **Performance**: `QueryPerformanceCounter`를 사용하여 나노초 단위의 정밀한 실행 시간을 측정합니다. 이는 멀티스레딩 최적화 전후의 성능 비교에 활용됩니다.
- **Timer**: 콜백 함수를 특정 시간 간격으로 실행하거나 정해진 횟수만큼 반복 실행하는 기능을 제공합니다.

## 3. 렌더링 시스템과의 관계
- 현재 렌더링 루프 자체는 단일 메인 스레드에서 Direct3D 11 Immediate Context를 사용하는 구조입니다.
- 멀티스레딩은 주로 데이터 로딩, 물리 연산 준비, 혹은 Compute Shader 실행 전후의 데이터 가공 등 메인 렌더링 스레드를 방해하지 않아야 하는 작업에 적합하도록 설계되어 있습니다.

## 4. 임계 영역 (Critical Section)
- 데모 코드에서 `m.lock()`과 `m.unlock()`을 명시적으로 호출하여 멀티스레드 환경에서의 안정성을 확보하는 패턴을 보여줍니다.
