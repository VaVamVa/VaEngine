### DeferredLighting Compute 셰이더 작성

**파일:** `Engine/_Shaders/DirectX/DeferredLighting.hlsl` (CS 단일 파일)

**바인딩 레이아웃:**

| 레지스터 | 이름 | 내용 |
|---|---|---|
| `b0` | `CB_DeferredCamera` | `InvViewProj`, `EyePos`, `ScreenW`, `ScreenH` |
| `b2` | *(Lighting.hlsli)* | `CB_Lights` — gDirLight / gPointLights / gSpotLights / gNumPointLights 등 |
| `t0` | `gAlbedoAO` | G-Buffer RT0 SRV |
| `t1` | `gNormalRough` | G-Buffer RT1 SRV |
| `t2` | `gMaterialBuf` | G-Buffer RT2 SRV |
| `t3` | `gDepth` | Depth SRV (`R24_UNORM_X8_TYPELESS`) |
| `u0` | `outHDR` | HDR 출력 UAV (`RWTexture2D<float4>`) |

**CB_DeferredCamera 레이아웃 (§12.6):**

```hlsl
cbuffer CB_DeferredCamera : register(b0)
{
    float4x4 InvViewProj;
    float3   EyePos;
    float    _pad;
    uint     ScreenW;
    uint     ScreenH;
    float2   _pad2;
};
```

**CSMain 흐름:**

1. `id.x >= ScreenW || id.y >= ScreenH` → 조기 종료
2. G-Buffer 4종 읽기 (depth, albedoAO, normalRough, metallic)
3. `depth >= 1.0` → 스카이 영역, `outHDR = (0,0,0,1)` 초기화 후 조기 종료  
   *(SkyPass가 이후 덮어쓰므로 Lighting 계산 생략)*
4. NDC → WorldPos 역투영:  
   `ndcPos.y = -ndcPos.y` (DX NDC Y축 처리)  
   `wPos = mul(ndcPos, InvViewProj)` → `worldPos = wPos.xyz / wPos.w`
5. Phong 조명 누적 (Directional + Point Lights + Spot Lights)  
   roughness → specPow: `max(1.0 - roughness, 0.01) × 128`
6. `outHDR[id.xy] = float4(result, 1.0)`

**Dispatch 파라미터:** `[numthreads(8, 8, 1)]` → `Dispatch(ceil(W/8), ceil(H/8), 1)`

**설계 결정:**
- Metallic은 G-Buffer에서 읽어 두지만 Phong 단계에서는 미사용 (추후 PBR 전환 시 활용)
- Lighting.hlsli를 `#include`하여 CB_Lights(b2) 재사용 — 별도 cbuffer 중복 정의 없음

**CMakeLists.txt:** `DeferredLighting_CS.cso` 컴파일 항목 + `CompileShaders` DEPENDS 추가 완료

#### 2. WO_Kachujin Mesh 미출력 (Gizmo는 정상)

**원인:**

Deferred 파이프라인에서 `ForwardPass` 가 호출되지 않으므로, `AnimationPass` 가 사용하는 트랜지언트 depth buffer 가 초기화(`Clear`)되지 않음.

DX12 Default heap 리소스는 0으로 제로 초기화 → depth = 0.0 (Near plane).
`LESS` 비교 함수에서 `fragment_depth < 0.0` 은 절대 성립하지 않으므로 모든 픽셀이 depth test 실패 → WO_Kachujin 완전 컬링.

Gizmo (DebugLine) PSO 는 `depthEnable = false` → depth test 무관, 정상 출력.

**근본 해결:** 단계 6에서 AnimationPass 자체를 제거하고 스키닝 메시를 GBufferPass에 통합.

**임시 수정 (단계 6 전까지 필요 시):** `AnimationRenderer.cpp` AnimationPass::Execute

```cpp
// 변경 전
passDesc.depthStencil.loadAction      = ELoadAction::Load;
passDesc.depthStencil.storeAction     = EStoreAction::DontCare;

// 변경 후
passDesc.depthStencil.loadAction      = ELoadAction::Clear;
passDesc.depthStencil.storeAction     = EStoreAction::Store;
passDesc.depthStencil.clearColor[0]   = 1.0f;
```

```
  4. DeferredLightingPass  — lightingRenderer.AddPasses
  5. DeferredBlitPass      — hdrOut SRV → backBuffer (Clear)
  6. AnimationPass         — animationRenderer.AddGraphicsPasses
  7. TransparentPass       — forwardRenderer.AddTransparentPasses
  8. DebugLinePass/TextPass
```

#### UpdateResourceStates

Execute 후 `graph.GetCurrentState(...)` 로 rt0/rt1/rt2/hdrOut/depth 상태를 기록 → 다음 프레임 ImportResource에 사용.

- `ForwardRenderer renderer` + `AnimationRenderer animationRenderer` → `SceneRenderer sceneRenderer`
- `#include "Render/ForwardRenderer.h"` / `AnimationRenderer.h` → `#include "Render/SceneRenderer.h"`

#### Execute.cpp 수정

**OnInitialize:** 6개 렌더러 초기화 호출 → `sceneRenderer.Initialize` + `InitializeGBuffer` / `InitializeLighting` / `InitializeBlit` / `InitializeSky` / `InitializeForward` / `InitializeAnimation` / `InitializeDebugLines` / `InitializeDebugText`

#### 알려진 제약사항

1. **Sky PSO 포맷 불일치**: `ForwardRenderer::InitializeSky` 의 `skyPsoDesc.rtvFormats` 가 `R8G8B8A8_UNORM` 이지만, `DeferredSkyPass` 는 `hdrOut` (`R16G16B16A16_FLOAT`) 에 렌더링. DX12 validation layer 경고 발생 가능 → `ForwardRenderer.cpp:394` 를 `R16G16B16A16_FLOAT` 로 수정 필요.

2. **Animation/Transparent Depth 격리**: `AnimationPass`·`TransparentPass`·`DebugLinePass` 는 트랜지언트 depth를 재사용하지만, Deferred 경로에서는 이 depth를 Clear하는 패스가 없음 → 이전 프레임 DontCare 결과가 로드됨. 시각적 아티팩트 가능성 있으나 크래시는 없음. (추후 shared depth 연동으로 해결 예정)

### 이슈 2: WO_Kachujin 메시 미출력 (기즈모는 출력됨)

**증상**: 실행 성공 후 큐브 등 정적 메시는 보이지만 Kachujin(스키닝 메시) 폴리곤이 전혀 보이지 않음. 기즈모(DebugLine)는 정상 출력.

**분석**:
- `DebugLine` PSO: `depthEnable = false` → depth test 없이 통과 → 보임
- `AnimationPass` depth: 트랜지언트 depth 버퍼, DX12 Default 힙 zero-init → depth 값 0.0
- `AnimationPass::Execute` `depthStencil.loadAction = ELoadAction::Load` → Deferred 이후 depth clear 없이 0.0 값 로드
- LESS 비교에서 0.0 depth → 모든 프래그먼트 실패 → Kachujin 불가시

**근본 원인**: Deferred 파이프라인으로 전환 후 공유 depth(1.0 clear)를 사용하지 않고, 별도 트랜지언트 depth(0.0 zero-init)를 `AnimationPass`가 Load하여 depth test 전부 실패.

**즉시 해결**: `AnimationPass::Execute` `clearColor[0] = 1.0f`, `loadAction = Clear`, `storeAction = Store` 변경 — 단, Step 6에서 `AnimationPass` 자체가 제거되므로 임시 조치.

**근본 해결**: Step 6 — 스키닝 메시를 `GBufferPass`(공유 depth, 1.0 clear)로 통합.
