## Q1: ExecAndroid에서 Engine/Application 참조 경고

**질문:**
ExecAndroid 프로젝트에서 Engine과 Application이 지원되지 않는 참조 항목이라는 경고가 뜸.

**답변:**

### 결론
정상 동작. ExecAndroid는 MSBuild 프로젝트 참조를 사용할 수 없는 구조이므로 경고가 맞으며, 무시해도 됨.

### 원인

| | ExecWindows | ExecAndroid |
|---|---|---|
| 빌드 시스템 | MSBuild | Gradle + Android NDK |
| 컴파일러 | MSVC (x64) | Clang (ARM/ARM64) |
| 참조 방식 | `<ProjectReference>` 가능 | MSBuild 프로젝트 참조 **불가** |

Engine / Application은 MSVC용 `.vcxproj` Static Library로, ARM용 Clang으로 컴파일해야 하는 Android에서는 MSVC `.lib`를 그대로 링크할 수 없음. VS가 이를 인식하여 경고를 표시하는 것.

### 해결 방향

**단기 (현재 — CMake 미도입):**
ExecAndroid의 참조 설정은 건너뜀. Windows 파이프라인(`ExecWindows → Application → Engine`) 먼저 완성.

**장기 (CMake 도입 후):**
```
Engine/CMakeLists.txt
Application/CMakeLists.txt
ExecAndroid/app/src/main/CMakeLists.txt  ← add_subdirectory로 포함
```
`app/build.gradle`에 아래 설정 추가 시, Gradle이 NDK Clang으로 Engine + Application을 함께 빌드:
```groovy
android {
    defaultConfig {
        externalNativeBuild {
            cmake { path "src/main/CMakeLists.txt" }
        }
    }
}
```

### 요약
- ExecAndroid의 Engine / Application 참조 경고는 정상이며 제거 불가
- Android 네이티브 연동은 CMake 전환 시점에 함께 진행
- 현재는 ExecWindows 참조 설정에 집중
