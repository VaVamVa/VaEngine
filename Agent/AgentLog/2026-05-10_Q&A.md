## Q1. 외부 HDR Asset 적용 방법

**Q**: Sky 렌더링이 그라디언트로 동작 확인. 외부 HDR Asset 적용 시 파일 형식과 경로를 알고 싶다.

**A**:

### 파일 형식

**`.hdr` (Radiance RGBE 포맷)** 만 지원된다.

`TextureFloat_DirectX::LoadFromFile`이 내부에서 `stbi_loadf()`를 호출하며, stb_image가 HDR 포맷으로 지원하는 확장자는 `.hdr`뿐이다. `.exr`은 stb_image 미지원.

다운로드 사이트: [Poly Haven](https://polyhaven.com/hdris) → **Equirectangular / .hdr** 선택.

### 경로 불일치 문제

당시 `ITexture::LoadFromFile` 시그니처가 `const wchar_t*`였고, `ASSETS_DIR`은 narrow string 매크로였다 — 두 타입이 불일치하여 직접 연결 불가능한 상태였다.

→ Q2에서 인터페이스 자체를 narrow string으로 변경하여 해결.

## Q2. LoadFromFile 인터페이스 개선 방향

**Q**: `VaEngine/_Assets/HDR/`에 HDR 파일 고정 배치 예정. `ASSETS_DIR`은 `"${CMAKE_SOURCE_DIR}/_Assets/"` (후행 슬래시 포함)로 두고, 개발자가 `LoadFromFile(device, ASSETS_DIR "HDR/filename.hdr")` 형태로 입력하게 하려면 어떻게 해야 하나?

**A**:

`ITexture::LoadFromFile` 시그니처를 `const wchar_t*` → `const char*`로 변경하는 것이 가장 단순하다.

**이유**:
- stb_image 함수(`stbi_load`, `stbi_loadf`)는 원래 `const char*`를 받는다.
- 기존 구현은 wchar_t를 WideCharToMultiByte로 변환 후 stbi에 넘기는 불필요한 단계가 있었다.
- `const char*`로 통일하면 변환 단계 제거, ASSETS_DIR 매크로와 문자열 리터럴 직접 연결 가능.

**변경 범위**:

| 파일 | 변경 내용 |
|---|---|
| `ITexture.h` | `wchar_t*` → `char*` |
| `Texture_DirectX.h/.cpp` | 시그니처 변경 + WideCharToMultiByte 제거 |
| `TextureFloat_DirectX.h/.cpp` | 시그니처 변경 + WideCharToMultiByte 제거 |
| `WorldModel.cpp` | `.wstring()` → `.string()` |
| `WorldAnimatedModel.cpp` | wstring 임시 변수 제거, texturePath.c_str() 직접 전달 |
| `Application/CMakeLists.txt` | ASSETS_DIR 후행 슬래시 추가 |
| `WO_Tower.cpp` | `"/Tower"` → `"Tower"` (중복 슬래시 제거) |
| `WO_Kachujin.cpp` | `"/Kachujin/"` → `"Kachujin/"` |

**사용 예시**:
```cpp
skyTexture->LoadFromFile(device, ASSETS_DIR "HDR/sky.hdr");
```
