# 2026-05-11 (세션 1) — 렌더링 파이프라인 리팩터링 및 멀티패스 구축

## 🛠️ 작업 요약
*   **통합 커맨드 스트림(Unified Command Stream)**: 정적 메쉬와 스킨드 메쉬를 하나의 `RenderCommand` 리스트로 통합 관리.
*   **자동화된 64비트 정렬 키(Automated SortKey)**: 카메라 거리에 따른 Depth 계산 및 텍스처 알파 채널 기반의 Translucent 판정 자동화.
*   **멀티패스 렌더링(Multi-pass Rendering)**: `ForwardPass`(불투명)와 `TransparentPass`(반투명) 분리 및 실행 순서(Z-Sorting) 보장.
*   **아키텍처 문서화**: 포트폴리오용 도식화 가이드 및 draw.io XML 논리 구조 문서 생성.

## 상세 내역

### 1. RenderScene & RenderCommand 통합
*   `RenderCommand` 구조체를 확장하여 `SkinnedMesh*`, `transformsMap` 등 애니메이션 관련 필드 통합.
*   `AddSkinnedMesh`가 별도 리스트가 아닌 공통 `commands` 리스트에 데이터를 적재하도록 수정.
*   **결과**: 카추진(애니메이션)과 타워/큐브(정적)가 하나의 리스트에서 통합 정렬 가능해짐.

### 2. 정렬 시스템 고도화 (SortKey)
*   `RenderScene::CalculateSortKey` 구현:
    *   `Matrix4x4`의 이동 행렬(`m[3][0..2]`)과 카메라 위치를 이용한 유클리드 거리 계산 자동화.
    *   `ITexture::HasAlpha()`를 통한 투명도 비트(`translucent`) 자동 설정.
    *   `uint8_t layer` 인자를 추가하여 수동으로 렌더링 우선순위를 지정할 수 있는 옵션 제공.

### 3. ForwardRenderer 멀티패스 리팩터링
*   `IRenderPass`를 상속받는 `ForwardPass`와 `TransparentPass`로 분리.
*   `ForwardRenderer::Render` 함수가 `isTransparentPass` 플래그에 따라 명령 리스트를 필터링하여 그리도록 수정.
*   **렌더링 순서**: `SkyPass` (ELoadAction::Clear) → `ForwardPass` (Opaque) → `TransparentPass` (Blended).

### 4. 컴파일 에러 해결 및 API 동기화
*   `ITexture`, `ITexture2DArray`, `ITextureUAV` 인터페이스에 `HasAlpha()` 순수 가상 함수 추가 및 DX12 구현체 반영.
*   `Math::Container`의 `Vector3`에 `static Distance()` 유틸리티 함수 추가.
*   `ForwardRenderer.h`의 함수 시그니처 동기화 및 `WO_Cube.cpp` 호출부 업데이트.
*   `AnimationRenderer` — 통합 커맨드 스트림에 맞게 API 동기화:
    *   `scene.GetSkinnedCommands()` → `scene.GetCommands()` 로 교체.
    *   `cmd.mesh` → `cmd.skinnedMesh` 필드 참조로 전환 (`Render` / `RenderCompute` 양쪽 모두 적용).
    *   `AddPasses` 내 unique mesh 추출 루프도 동일하게 수정.

### 5. RenderGraph 디버그 개선
*   Transient Depth 신규 생성 시 `VA_LOG("RenderGraph", ...)` 추가 (크기 정보 출력).
*   Barrier 상세 로그(`Resource[ptr] A→B`)를 주석 처리 — 매 프레임 출력되어 패널 노이즈 유발.
*   `VA_DRAW_PANEL(2, ...)` 로 현재 프레임의 **패스 수 / 배리어 수** 실시간 표시.

### 6. CameraManager API 확장
*   `GetPosition()` / `GetYaw()` / `GetPitch()` 접근자 추가 (`CameraManager.h/.cpp`).
*   내부적으로 `FreeCamera` 위임 호출.

### 7. VaProgramName 디버그 강화
*   초기화 단계별 `VA_LOG` 추가: 에셋 로드(Kachujin 클립 수, WO_Cube, WO_Tower, HDRi), 라이트 활성화.
*   `VA_DRAW_PANEL(3)` 카메라 월드 위치, `VA_DRAW_PANEL(4)` Yaw/Pitch(도 단위) 실시간 표시.
*   `VA_DRAW_PANEL(6)` 현재 재생 중인 클립 이름 표시 (`GetClipName(clipIndex)`).
*   클립 자동 전환 타이머: `PlayBlend` (3클립 가중 합성) → `PlayTween` (Tween 방식)으로 복귀.
    *   `PlayBlend` 호출은 주석으로 보존.

### 8. 문서화 작업 (`Agent/AgentLog/`)
*   `Debugging_Locations.md`: 엔진 내 로그 및 패널 출력 위치 정리.

## 💡 특이 사항 및 기술적 성과
*   **Z-Sorting & Early-Z**: 불투명 객체는 가까운 순서로(Front-to-Back), 반투명 객체는 먼 순서로(Back-to-Front) 정렬되는 논리를 완벽히 구현.
*   **Data-Driven Pipeline**: 개발자가 직접 키를 관리하지 않아도 엔진이 리소스를 분석해 최적의 패스로 분배하는 자동화 아키텍처 달성.
*   **Troubleshooting**: DX12의 하드웨어 Depth Test 제약 사항과 소프트웨어 정렬 사이의 관계를 명확히 규명하고 레이어 시스템으로 대안 마련.
