#### 2. 첫 렌더링 명령 완성

`Execute::OnRender` 전체 흐름 구현 및 단색 출력 확인:

- `ClearRenderTargetView` — 지정 색상 클리어
- `ResourceBarrier` RenderTarget → Present
- `CommandQueue::ExecuteCommandLists` → `SwapChain::Present`

#### 3. GPU 동기화 안정화

- Fence 카운팅 방식으로 CPU-GPU 실행 순서 보장
- `CommandAllocator` 리셋 에러 해결

#### 4. CommandAlloc / CommandList 구현 세부

| 항목 | 내용 |
|---|---|
| `CommandAlloc::Register` 버그 수정 | `if` 블록 성공 경로에 `return` 누락 → 항상 throw 발생하던 문제 수정 |
| `CommandAlloc::Reset` | `commandAllocator->Reset()` + HRESULT 검사 |
| `CommandList::Begin` | `commandList->Reset(allocHandle, nullptr)` — DX12 Reset이 Begin에 해당 |
| `CommandList::Close` | `commandList->Close()` + HRESULT 검사 |

#### 5. 컴파일 에러 진단 및 해결

| 에러 | 원인 | 해결 |
|---|---|---|
| `C4430` IRenderDevice.h | `ICommandAlloc` 파싱 실패로 인한 연쇄 에러 (`[[nodiscard]]` 문제 아님) | 아래 C2027 해결 후 소멸 |
| `C4150 / C2027 / C2338` | `ExecWindows.cpp` TU에서 `unique_ptr<ICommandAlloc>` 소멸 시 불완전 타입 | `Execute.h`에 `~Execute()` 명시 선언, `Execute.cpp`에서 `= default` 정의 |
| `C6001 / C26827` (DX 헤더) | MSVC Code Analysis의 서드파티 헤더 오탐(false positive) | CMake `SYSTEM` 키워드로 외부 헤더 경고 억제 |
| `E3364` SwapChain_DirectX | `TRHIResource<ID3D12Resource>` 값 타입에 `->` 적용 | `.` 접근 또는 포인터 방식으로 수정 |

#### 6. IRHIResource 템플릿 설계

| 항목 | 결론 |
|---|---|
| Public `IRHIResource` | 비 템플릿 유지 — `ResourceBarrier*`에서 다형성 사용 |
| DX12 구현체 | `TRHIResource<TDXResource>` 템플릿 — alias로 타입별 확장 |
| 메모리 오류 | 종료 시 COM 객체 `Release` 보장을 위해 커스텀 삭제자 적용 |
